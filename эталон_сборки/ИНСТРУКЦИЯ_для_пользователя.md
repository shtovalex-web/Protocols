# Инструкция по работе с программой (протоколы проверки знаний по охране труда)

Программа **ProtocolOOT** заполняет шаблон протокола (Word или текстовый файл) данными о сотрудниках и программах обучения из Excel, позволяет сохранить результат в **DOCX** и **PDF** и вести **журнал** записей. Есть выгрузка для **Минтруда / АКОТ**.

Подробные ответы на вопросы — в файле **FAQ.md** и в меню **«Справка» → «Справка и FAQ»**.

---

## 1. Запуск программы

- **Вариант с установленным Python:** в папке с программой выполните `py -3 main.py` или запустите **`запуск.bat`** (в эталонной копии проекта).
- **Вариант с одним файлом:** запустите **`ProtocolOOT.exe`**. Рядом с exe должны лежать шаблоны и файлы Excel, которые поставляются вместе со сборкой (см. папку вывода PyInstaller и файл **`ИНСТРУКЦИЯ_папки_сборки.txt`**).

Рабочие файлы (база **`protocols.db`**, папки сохранения и т.д.) создаются **в той же папке**, что и программа (или exe).

---

## 2. Что нужно подготовить в Excel

1. **Сотрудники** — обычно файл **`Data_base.xlsx`** с листом сотрудников (допустимые имена листа: `rabotnik`, «Работники», «Сотрудники» и др.; первая строка — заголовки).
2. **Программы обучения** — либо отдельный файл **`Programs_base.xlsx`**, либо те же листы в **`Data_base.xlsx`**:
   - лист **B** — программа «Б»;
   - **V_PROF** — должности и фрагменты для блоков «Б», «ПП», «СИЗ», «В»;
   - **PP**, **SIZ** — блоки первой помощи и СИЗ;
   - **V** — реестр программ (сопоставление с текстами и ID для Минтруда).

Настроить пути к файлам можно в **«Администрирование» → «Настройки и данные…»**.

---

## 3. Главное окно: порядок работы

1. **Обновить базы с диска** — перечитать Excel с диска и обновить внутренний кэш (полезно после правок в файлах).  
   **Горячая клавиша: F5.**

2. **Поиск** — фильтр по списку сотрудников (ФИО, должность, подразделение, СНИЛС).  
   **Ctrl+F** — быстрый переход в поле поиска.

3. **Список сотрудников** — выделите одного или нескольких (**Ctrl**, **Shift**). При выборе одной строки должность и подразделение подставляются в поля ниже.

4. Если человека **нет в списке**, введите **ФИО**, при необходимости **должность** и **подразделение** вручную.

5. **Программы обучения** — отметьте галочками нужные блоки (**Б**, **ПП**, **СИЗ**, **В**). Для шаблона **Word** нужна **хотя бы одна** программа (если не используете только текстовый шаблон). Наведите курсор на галочку — краткая подсказка, откуда берутся данные.

6. **Дата** и **№ протокола** — в бланк Word номер часто подставляется в формате **номер–месяц–год** по полю даты (см. также «Переменные шаблона» в настройках).

7. **Оценка** и **Проверка знаний** (плановая / внеплановая) — для таблицы результатов в протоколе.

8. Внизу окна — **строка состояния**: сколько сотрудников загружено и сколько показано при поиске.

---

## 4. Формирование протокола и предпросмотр

1. Нажмите **«Сформировать протокол»**.
2. Откроется отдельное окно **«Предпросмотр протокола»** с текстом (для Word — с упрощённым отображением полужирного и курсива). Его можно закрыть и открыть снова кнопкой **«Предпросмотр…»** на главном окне.
3. Запись о протоколе при этом может сохраниться в **журнал** (база **`protocols.db`**); полный текст протокола в журнал **не кладётся** намеренно (меньше персональных данных в одном файле).

---

## 5. Сохранение в DOCX и PDF

- Кнопки **«Сохранить в DOCX»** и **«Сохранить в PDF»** есть **на главном окне** и **в окне предпросмотра**.
- По умолчанию предлагается папка **`Protokol`** рядом с программой (если она создана).
- **PDF «как в Word»** на Windows: нужен установленный **Microsoft Word** и корректная работа COM. Если Word недоступен, программа может сохранить **упрощённый PDF** из текста предпросмотра (без сложного оформления таблиц).

---

## 6. Отдельный файл на каждого сотрудника

Кнопка **«По одному на каждого → в папку…»** (только для шаблона **Word**):

- для каждого выбранного сотрудника формируется свой **DOCX** и своя запись в журнале;
- номера протоколов идут по порядку от числа в поле **«№ протокола»**;
- после выполнения в предпросмотре показывается **последний** сформированный документ.

---

## 7. Меню «Администрирование»

| Пункт | Назначение |
|--------|------------|
| **Настройки и данные…** | Файлы сотрудников и программ, шаблон протокола, реестр обученных для Минтруда, выгрузка шаблонов восстановления, справка по переменным шаблона. |
| **Приказ и комиссия…** | Состав комиссии и данные для протокола (в т.ч. родительный падеж при установленных словарях pymorphy2). |
| **Выгрузка шаблонов восстановления…** | Копия пустых/эталонных файлов в выбранную папку на случай потери данных. |

---

## 8. Меню «Минтруд»

- **Реквизиты работодателя** — для выгрузки в шаблон XSD.
- **Файл реестра обученных (.xlsx)** — выгрузка с портала; по ней можно подставлять регистрационные номера в протокол (см. FAQ по правилам совпадения ФИО/СНИЛС и должности для программы «Б»).
- **Пересформировать протокол (номера реестра)** — пересборка с теми же полями после смены файла реестра.
- **Шаблон для загрузки на сайт…** — заполнение официального Excel **`Шаблон_Минтруд_XSD_УМН.xlsx`** по журналу; при сохранении рядом создаётся файл **`.xml`** с теми же данными для контроля.

---

## 9. Журнал протоколов

В настройках есть переход к **журналу** (список сохранённых записей из **`protocols.db`**). Можно открыть краткие сведения по записи; при наличии сохранённого текста в базе — **«В окно предпросмотра»** для повторного сохранения в файл.

---

## 10. Горячие клавиши

| Клавиши | Действие |
|---------|----------|
| **F5** | Обновить базы с диска |
| **Ctrl+F** | Фокус в поле поиска по списку |
| В списке сотрудников **Ctrl** / **Shift** | Множественный выбор |

Полный список — **«Справка» → «Горячие клавиши…»**.

---

## 11. Если что-то не работает

1. Проверьте, что **Excel** не открыт с **блокировкой** файла только для чтения в сети, и нажмите **F5**.
2. Откройте **FAQ.md** или **«Справка и FAQ»**.
3. При ошибках смотрите файл **`protocol_errors_journal.txt`** в папке программы.
4. Для **.exe**: переносите **всю папку** со сборкой, не один только файл exe.
