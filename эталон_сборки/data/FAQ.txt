# Справка и FAQ — программа для формирования протоколов проверки знаний по охране труда

Ключевые слова: сотрудники Excel Data_base шаблон протокол Word DOCX PDF журнал Минтруд реестр СНИЛС программа Б В ПП СИЗ комиссия выгрузка

**Пошаговая инструкция по каждому блоку окна (простым языком):** **ПОДРОБНАЯ_ИНСТРУКЦИЯ_для_пользователя.docx** (или .md) в папке программы (рядом с FAQ.md).

---

## 1. С чего начать

**Вопрос: что нужно для работы?**  
Файл с сотрудниками (обычно `Data_base.xlsx`), шаблон протокола Word (`default_protocol.docx` или свой), при необходимости — листы программ в том же Excel (B, V_PROF, PP, SIZ, лист V).

**Вопрос: где настроить файл сотрудников?**  
Меню «Администрирование» → «Настройки и данные…» → кнопка «Файл сотрудников…» или положите `Data_base.xlsx` в папку с программой.

**Вопрос: список сотрудников пуст**  
Нажмите «Загрузить из Excel». Проверьте имя листа (рабочие варианты: rabotnik, работники, сотрудники и т.п.) и строку заголовков в первой строке.

---

## 2. Формирование протокола

**Вопрос: как выбрать несколько человек?**  
В списке сотрудников используйте Ctrl или Shift для множественного выбора. Строки «▾ Подразделение (N)» — заголовки групп: щёлкните, чтобы свернуть или развернуть список.

**Вопрос: поиск в списке**  
Поле «Поиск» фильтрует по ФИО, должности, подразделению (и связанным полям). Ввод с небольшой задержкой, чтобы не тормозить при наборе.

**Вопрос: совмещения и программа «В»**  
Если у сотрудника в Excel несколько должностей и отмечена «В», при формировании откроется диалог: отметьте должности для «В» и выберите основную для графы «Должность» (у каждого ФИО отдельно). Подробно — **ПОДРОБНАЯ_ИНСТРУКЦИЯ**, §11.

**Вопрос: что отметить в «Программах обучения»?**  
Б — лист B; В — V_PROF по должности; PP и СИЗ — из кэша листов. Для шаблона Word нужна хотя бы одна отмеченная программа (если не используете только текстовый шаблон).

**Вопрос: номер протокола в бланке**  
В шаблон подставляется формат номер–месяц–год (месяц и год берутся из поля «Дата»).

**Вопрос: сохранение**  
«Сохранить в DOCX» / «Сохранить в PDF» — после предпросмотра. PDF с оформлением Word (Windows): установленный Microsoft Word и пакет **pywin32** (`pip install pywin32`); программа вызывает Word напрямую, без docx2pdf (из‑за ошибки прогресс-бара в окне без консоли).

## 3. Блок программы «В» (V_PROF)

**Вопрос: нумерация 2., 3. в таблице**  
Если в ячейке V_PROF строка начинается с `2.` или `3)` и далее текст программы, в шапке блока «В» и в ячейке результата будет тот же номер (не всегда с 1).

**Вопрос: сопоставление с реестром**  
Для поиска в файле реестра из текста строки убирается ведущий номер; сравнивается наименование программы.

## 4. Минтруд и реестр обученных

**Вопрос: выгрузка на сайт**  
Меню «Минтруд» → «Шаблон для загрузки на сайт…». Нужен файл `Шаблон_Минтруд_XSD_УМН.xlsx` в папке программы. Реквизиты работодателя — в том же меню. При сохранении выгрузки создаются два файла с одним именем: `.xlsx` (официальный шаблон) и `.xml` с теми же строками данных (удобно для контроля и обработки; формат портала уточняйте по актуальной инструкции АКОТ).

**Вопрос: программа «В» в шаблоне Минтруда**  
Для каждой записи журнала с программой «В» в выгрузку попадают **все** наименования программ, как в сформированном протоколе (по каждой должности из протокола — отдельная строка). Колонка «Должность» для программы «В» берётся **из протокола** (не из Excel сотрудников). Для программ «Б», ПП и СИЗ должность по-прежнему из файла сотрудников.

**Вопрос: номера из реестра в протоколе**  
После регистрации проверки скачайте Excel «Реестр обученных…». Укажите файл в настройках («Реестр обученных…») или «Минтруд» → «Файл реестра обученных».  
Подстановка: СНИЛС (приоритет) или ФИО, номер протокола, программа блока. Строгое совпадение колонки «Должность» в реестре с должностью сотрудника действует **только для блока программы «Б»** (чтобы при одном ФИО/СНИЛС и разных должностях не дублировался один №). Для ПП, СИЗ и «В» должность не обязательна для подстановки; при совпадении программы она лишь помогает выбрать строку при нескольких вариантах. Без должности в строке сотрудника фильтр по должности для «Б» не применяется.

**Вопрос: структура файла реестра**  
Ожидаются колонки «Номер в реестре», ФИО (одна колонка или «Фамилия», «Имя», «Отчество»), желательно СНИЛС, программа, номер протокола.

**Вопрос: ручной номер реестра**  
Поле «Регистрационный номер» в настройках заполняет пробелы, если из файла номер не подобран.

## 5. Комиссия и приказ

**Вопрос: где задать состав комиссии?**  
«Администрирование» → «Приказ и комиссия…». Кандидаты подтягиваются с листа `komission` в файле Excel (см. подсказки в окне).

**Вопрос: несколько комиссий для разных подразделений (с 1.5)**  
Вверху окна — поле **«Сохранённая комиссия (название / подразделение)»**, например «НПС и ЦТТ». Введите название, заполните приказ и состав, нажмите **«Сохранить приказ и состав в базу данных»**. Профили хранятся в `protocols.db`; для охраны труда и технических протоколов — отдельные списки. Кнопки **«Загрузить»** и **«Удалить профиль»** — работа с сохранёнными вариантами.

## 6. Журнал протоколов

**Вопрос: где история?**  
«Настройки и данные…» → «Журнал (охрана труда)…» или «Журнал (тех. протоколы)…». Полный текст протокола в базе не хранится — только ФИО, дата, №, оценка. Кнопка **«Выгрузить реестр…»** в окне журнала — Excel/CSV со списком сформированных протоколов (на нескольких человек — отдельная строка на каждое ФИО).

**Вопрос: дублируются записи в журнале (с 1.5)**  
При повторном формировании протокола с тем же **№, датой и видом** запись **обновляется**, а не добавляется. При первом запуске после обновления программа может удалить старые дубликаты из базы (сообщение один раз за версию). В списках журнала и выгрузки Минтруд показывается одна актуальная строка на протокол.

**Вопрос: что нового в версии программы?**  
«Справка» → **«Журнал доработок…»** или файл `ЖУРНАЛ_ДОРАБОТОК.md` в папке комплекта.

## 7. Неполадки

**Вопрос: таблица в Word не заполнилась**  
В шаблоне должна быть таблица с маркерами вида 1.1, 2.1 и текстом «ФИО» во второй колонке — см. «Переменные шаблона» в настройках.

**Вопрос: реестр не читается**  
Проверьте первый лист, заголовки и наличие колонок «Номер в реестре» и ФИО.

**Вопрос: openpyxl**  
Для Excel-операций нужен пакет openpyxl (`pip install openpyxl`).
