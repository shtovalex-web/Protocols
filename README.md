# Протокол проверки знаний

Desktop-приложение на Python/Tkinter для формирования протоколов проверки знаний по шаблону Word.

Программа загружает сотрудников из Excel, подставляет данные в `default_protocol.docx`, формирует таблицу результатов проверки знаний, добавляет сведения о приказе и комиссии, сохраняет протокол в DOCX/PDF и ведет локальный журнал в SQLite.

## Установка

1. Установите Python 3.10 или новее.
2. В папке проекта установите зависимости:

   ```bash
   pip install -r requirements.txt
   ```

3. Запустите приложение:

   ```bash
   python main.py
   ```

## Файлы рядом с программой

- `default_protocol.docx` - шаблон протокола по умолчанию.
- `Data_base.xlsx` - Excel-файл с сотрудниками и комиссией; локальный файл не хранится в Git.
- `protocols.db` - локальная база журнала протоколов и настроек комиссии.
- `last_protocol_no.json` - сохраненный последний введенный номер протокола.

Если файл сотрудников называется иначе, выберите его в меню **Администрирование -> Настройки и данные... -> Файл сотрудников...**.

## Структура Excel-файла

### Лист сотрудников

Основной лист: `rabotnik`. Также распознаются похожие имена: `работник`, `работники`, `сотрудники`, `сотрудник`, `список сотрудников`, `кадры`.

Первая строка должна содержать заголовки. Программа ищет колонки:

- ФИО сотрудника;
- должность / профессия;
- подразделение;
- совмещаемая или вторая должность, если есть.

Если заголовки не распознаны, используются первые три колонки: ФИО, должность, подразделение.

### Лист комиссии

Лист: `komission` или похожее имя (`комиссия`, `комиссия по проверке`, `commission`).

Данные читаются с 3-й строки:

- колонка A - ФИО председателя;
- колонка B - должность председателя;
- колонка D - ФИО члена комиссии;
- колонка E - должность члена комиссии.

### Листы программ

- `B` - название программы Б берется из первой непустой ячейки столбца 2, начиная со 2-й строки.
- `V_PROF`:
  - столбец A - должность;
  - столбец 2 - текст программы Б для шапки после фразы о проверке;
  - столбец 3 - первая помощь;
  - столбец 4 - СИЗ;
  - столбцы 5-19 - фрагменты программы В.

Для программы В фрагменты подбираются по должности сотрудника. Если у сотрудника есть совмещаемая должность, фрагменты по двум должностям объединяются без дублей.

## Быстрый порядок работы

1. Запустите `python main.py`.
2. Откройте **Администрирование -> Настройки и данные...**.
3. Выберите Excel-файл сотрудников или оставьте `Data_base.xlsx` рядом с программой.
4. При необходимости выберите другой шаблон DOCX.
5. Откройте **Администрирование -> Приказ и комиссия...**.
6. Укажите номер и дату приказа, выберите председателя и членов комиссии, затем сохраните настройки.
7. В главном окне выберите одного или нескольких сотрудников.
8. Отметьте нужные программы обучения: Б, первая помощь, СИЗ, В.
9. Проверьте дату, номер протокола, оценку и тип проверки знаний.
10. Нажмите **Сформировать протокол**.
11. Проверьте предварительный просмотр и сохраните результат в DOCX или PDF.

## Номер протокола

В поле **№ протокола** вводится базовый номер. В шаблон он попадает в виде:

```text
<номер>-<месяц>-<год>
```

Например, при номере `12` и дате `25.03.2026` будет подставлено `12-3-26`.

Если дата не распознана как `ДД.ММ.ГГГГ` или `ДД.ММ.ГГ`, в шаблон попадет только введенный номер.

## Шаблон протокола

Шаблон может быть DOCX или TXT, но основной режим работы рассчитан на DOCX.

Программа не требует переменных вида `{{ФИО}}` для основных полей. Она ищет типовые фрагменты бланка:

- строку с `ПРОТОКОЛ №` - для номера протокола;
- строку вида `«__» ___________ 20__ г.` - для даты;
- абзац о комиссии и приказе - для номера/даты приказа и состава комиссии;
- длинную строку подчеркиваний с запятой в конце - для темы/программ;
- таблицу результатов проверки знаний с маркером ФИО - для перестроения строк сотрудников.

Подробные правила доступны в приложении: **Администрирование -> Настройки и данные... -> Переменные шаблона**.

### Плейсхолдеры подписей комиссии

В конце DOCX-шаблона можно добавить:

- `{{ПРЕДСЕДАТЕЛЬ}}` или `{{CHAIR}}`;
- `{{ЧЛЕНЫ_КОМИССИИ}}` или `{{MEMBERS}}`.

Они заменяются на председателя и членов комиссии из окна **Приказ и комиссия...**.

## Журнал протоколов

После формирования протокол сохраняется в локальную базу `protocols.db`.

Журнал доступен через **Администрирование -> Настройки и данные... -> Журнал протоколов в базе**. В журнале можно посмотреть записи, загрузить текст в предварительный просмотр или очистить журнал.

## Частые проблемы

- **Не найден файл Excel** - положите `Data_base.xlsx` рядом с программой или выберите файл вручную.
- **Не найден лист сотрудников** - проверьте имя листа и заголовки первой строки.
- **Комиссия не подставляется** - заполните и сохраните окно **Приказ и комиссия...**, а в шаблоне оставьте абзац с упоминанием комиссии, приказа, председателя или членов.
- **Программа В пустая** - проверьте совпадение должности сотрудника со столбцом A листа `V_PROF` и наличие тем в столбцах 5-19.
- **PDF с некорректными русскими символами** - сначала сохраните DOCX; для PDF может потребоваться доступный системный шрифт с кириллицей.
