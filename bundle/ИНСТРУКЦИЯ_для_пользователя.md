# Инструкция по работе с программой (протоколы проверки знаний по охране труда)

Программа **ProtocolOOT** заполняет шаблон протокола (Word или текстовый файл) данными о сотрудниках и программах обучения из Excel, позволяет сохранить результат в **DOCX** и **PDF** и вести **журнал** записей. Есть выгрузка для **Минтруда / АКОТ**.

---

## Какую инструкцию читать

| Документ | Для кого |
|----------|----------|
| **ПОДРОБНАЯ_ИНСТРУКЦИЯ_для_пользователя.docx** (или .md в папке программы) | Пошагово **по каждому блоку окна** — максимально простым языком |
| **FAQ.md** | Ответы на отдельные вопросы; меню **«Справка» → «Справка и FAQ…»** |
| **ИНСТРУКЦИЯ_оформление_протоколов_Минтруд.md** | Цепочка с реестром и выгрузкой на сайт Минтруда |

---

## 1. Запуск программы

- **Вариант с установленным Python:** в папке с программой выполните `py -3 main.py` или запустите **`запуск.bat`** (в эталонной копии проекта).
- **Вариант с одним файлом:** запустите **`ProtocolOOT.exe`**. Рядом с exe — подпапка **`data/`** (шаблоны, справка, инструкции) и при необходимости файлы Excel в **корне** (см. **`ИНСТРУКЦИЯ_папки_сборки.txt`**).

Рабочие файлы (база **`protocols.db`**, папки сохранения и т.д.) создаются **в той же папке**, что и программа (или exe).

---

## 2. Что нужно подготовить в Excel

Файлы могут быть в формате **`.xlsx`** (рекомендуется) или **`.ods`** (LibreOffice). Программа при необходимости конвертирует ODS через **LibreOffice** (должен быть установлен). В папке комплекта могут лежать образцы **`Data_base.xlsx`** / **`Data_base.ods`**, **`Programs_base.xlsx`** / **`Programs_base.ods`**.

1. **Сотрудники** — обычно файл **`Data_base.xlsx`** с листом сотрудников (допустимые имена листа: `rabotnik`, «Работники», «Сотрудники» и др.; первая строка — заголовки).
2. **Программы обучения** — либо отдельный файл **`Programs_base.xlsx`**, либо те же листы в **`Data_base.xlsx`**:
   - лист **B** — программа «Б»;
   - **V_PROF** — должности и фрагменты для блоков «Б», «ПП», «СИЗ», «В»;
   - **PP**, **SIZ** — блоки первой помощи и СИЗ;
   - **V** — реестр программ (сопоставление с текстами и ID для Минтруда).

Настроить пути к файлам можно в **«Администрирование» → «Настройки и данные…»**.

## 3. Главное окно: порядок работы

**Расположение (с 1.5):** сверху — **поиск**; слева — **список сотрудников**; справа — поля **участника** и **реквизитов протокола**, ниже — блок **«Программы обучения»**.

1. **Обновить базы с диска** — перечитать Excel с диска (**F5**).
2. **Поиск** — фильтр по списку сотрудников (**Ctrl+F** — в поле поиска).
3. **Подразделение** — над списком; подставляется из Excel, можно изменить.
4. **Список сотрудников** — группы по подразделениям (щёлк по заголовку «▾ …» — свернуть/развернуть); выделите одного или нескольких (**Ctrl**, **Shift**).
5. При необходимости введите **ФИО**, **должность** вручную; для программы **«В»** при совмещениях — диалог выбора должностей.
6. **Программы обучения** — отметьте галочками нужные блоки (**Б**, **ПП**, **СИЗ**, **В**). Для Word нужна **хотя бы одна** программа.
7. **Дата** и **№ протокола** — в бланк Word номер часто в формате **номер–месяц–год** по полю даты.
8. **Оценка** и **Проверка знаний** — для таблицы результатов.
9. **«Сформировать протокол»** → **«Сохранить в DOCX»** / **PDF** (готовый DOCX можно сразу править в Word).

Подробное описание **каждой кнопки и поля** — в **ПОДРОБНАЯ_ИНСТРУКЦИЯ_для_пользователя.md**.

## 4. Меню (кратко)

| Меню | Назначение |
|------|------------|
| **Вид** | Цветовая схема окна (сохраняется в настройках) |
| **Администрирование** | Настройки файлов, комиссия, резервные копии шаблонов |
| **Минтруд** | Реестр обученных, реквизиты, выгрузка XSD на сайт |
| **Справка** | FAQ, **проверка обновлений** (.exe), журнал доработок, горячие клавиши, о программе |

---

## 5. Горячие клавиши

| Клавиши | Действие |
|---------|----------|
| **F5** | Обновить базы с диска |
| **Ctrl+F** | Фокус в поле поиска по списку |
| **Ctrl+V** | Вставка в поля ввода; **ПКМ** — меню «Вставить» |

Полный список — **«Справка» → «Горячие клавиши…»**.

---

## 6. Если что-то не работает

1. Проверьте, что **Excel** не блокирует файл, и нажмите **F5**.
2. Откройте **FAQ.md** или **«Справка и FAQ»**.
3. Прочитайте раздел «Частые ошибки» в **ПОДРОБНАЯ_ИНСТРУКЦИЯ_для_пользователя.md**.
4. При ошибках смотрите **`protocol_errors_journal.txt`** в папке программы.
5. Для **.exe**: переносите **всю папку** (exe + **data/**), не один только файл exe. Обновление с шары (с 1.5.2) — **«Справка» → «Проверить обновления…»**.
