# Инструкция: оформление протоколов от запуска до сохранения (с учётом данных Минтруда)

Документ описывает работу в **готовой программе ProtocolOOT** (заголовок окна: *ProtocolOOT — протоколы ОТ*) — от первого запуска до получения **заполненных** файлов протокола (**Word** и при необходимости **PDF**), включая **реестр обученных** с портала Минтруда / АКОТ и **выгрузку Excel + XML** для загрузки данных по шаблону XSD.

**Как получить PDF от этой инструкции:** откройте **ИНСТРУКЦИЯ_оформление_протоколов_Минтруд.docx** в Microsoft Word → **Файл → Сохранить как → PDF**.

---

## Часть 1. Что должно быть на компьютере

1. **Папка с программой**  
   - Вариант **exe:** вся папка со **`ProtocolOOT.exe`** и файлами комплекта (шаблоны Word, Excel, XSD-шаблон Минтруда — **в одной папке с exe**, как после сборки PyInstaller).  
   - Вариант **Python:** корень проекта с **`main.py`**, подпапка **`bundle/`** с `default_protocol.docx` и шаблоном Минтруда (программа сама находит каталог комплекта).

2. **Файл сотрудников** — обычно **`Data_base.xlsx`** (в папке данных программы или путь в **«Администрирование» → «Настройки и данные…»**).  
   На листе сотрудников — ФИО, должность, подразделение, **СНИЛС** (желательно для сопоставления с реестром Минтруда). Список сотрудников программа **читает из Excel**; в SQLite он не дублируется как основная «база сотрудников».

3. **Справочник программ** — отдельный **`Programs_base.xlsx`** или листы **B**, **V_PROF**, **PP**, **SIZ**, **V** в **`Data_base.xlsx`** или в отдельной книге (настройка «Файл программ»).

4. **Шаблон протокола** — **`default_protocol.docx`** или свой **.docx** / **.txt** (**«Администрирование» → «Настройки и данные…»** → выбор шаблона).

5. **Для PDF с оформлением как в Word (Windows):** установленный **Microsoft Word** и пакет **pywin32** (при запуске из Python: `pip install pywin32`; в типичной сборке **.exe** он уже включён). Программа вызывает Word через COM, без отдельного `docx2pdf`. Если Word недоступен, доступен **упрощённый PDF** из текста предпросмотра.

6. **Опционально:** **родительный падеж** для подписей комиссии — **pymorphy2** (см. `requirements.txt`). **Значок окна** — **`icon.ico`** в каталоге комплекта (`bundle/` при исходниках или рядом с **.exe**).

7. **Файл с портала Минтруда (при необходимости):**  
   Выгрузка **реестра обученных** в формате **Excel (.xlsx)** — после регистрации проверок. Точные шаги на сайте — только в **актуальной инструкции АКОТ / Минтруда** для работодателя.

8. **XSD-шаблон для выгрузки на сайт** должен быть в **каталоге комплекта** программы. Имя по умолчанию: **`Шаблон_Минтруд_XSD_УМН.xlsx`** (лист **«Шаблон»**). Допустимы также имена из поставки: **`!! Шаблон_Минтруд_XSD_УМН _ общ+.xlsx`**, **`Шаблон_Минтруд_XSD_УМН _ общ+.xlsx`**.

---

## Часть 2. Запуск программы

1. Запустите **`ProtocolOOT.exe`** или **`py -3 main.py`** из корня проекта (после `pip install -r requirements.txt`).  
2. Окно при старте может на мгновение не показываться — это нормально.  
3. Рядом с программой (каталог данных приложения) появляются **`protocols.db`** (SQLite: журнал, кэш программ, настройки), **`last_protocol_no.json`**, папки **`Protokol/`**, **`Mintrud/`** — не удаляйте без необходимости.  
4. Ошибки интерфейса пишутся в **`protocol_errors_journal.txt`** в том же каталоге данных.

---

## Часть 3. Настройка путей к файлам

1. **«Администрирование» → «Настройки и данные…»**.  
2. **«Файл сотрудников»** — **`Data_base.xlsx`** (или ваш файл).  
3. **«Файл программ»** — при отдельной книге укажите **`Programs_base.xlsx`**; если программы в **`Data_base.xlsx`**, отдельный файл может не понадобиться.  
4. **Выбор шаблона протокола** — при необходимости свой **.docx** / **.txt**.  
5. Нажмите **«Обновить базы с диска»** в главном окне или клавишу **F5**, чтобы обновились список сотрудников и кэш листов программ в SQLite.  
6. **«Переменные шаблона»** — как в документе размечены номер протокола, дата и таблица результатов.  
7. Поле **«Поиск»** по списку сотрудников: горячая клавиша **Ctrl+F** (фокус в поле поиска).

---

## Часть 4. Данные Минтруда: реестр обученных

Реестр нужен, чтобы в протокол **подставлялись регистрационные номера** из официальной выгрузки (после регистрации проверки знаний на портале). **Реестр не используется**, если включён режим **«Технический протокол»** (см. часть 6).

### 4.1. Получение файла на портале

1. Личный кабинет работодателя на портале **Минтруда / АКОТ** (адрес и меню — в официальной справке).  
2. Раздел **реестра обученных** (выгрузка с **номерами в реестре**, **ФИО**, **СНИЛС**, **программой**, **номером протокола**).  
3. Сохраните файл **Excel (.xlsx)** на диск.

### 4.2. Подключение файла в программе

1. **«Минтруд» → «Файл реестра обученных (.xlsx)…»** или тот же файл в **«Настройки и данные…»** (блок *Реестр обученных*).  
2. Укажите скачанный **.xlsx**.  
3. На первом листе должны быть колонки вроде **«Номер в реестре»**, **ФИО** (или фамилия, имя, отчество), по возможности **СНИЛС**, программа, номер протокола (подробнее — **FAQ** в меню **«Справка»** или файл **FAQ.md** в комплекте).

### 4.3. Как подставляются номера

- Сопоставление: **СНИЛС** (предпочтительно), **ФИО**, **номер протокола** и программа в выгрузке.  
- Для программы **«Б»** при нескольких строках на одного человека может учитываться **строгое совпадение должности**. Для **ПП**, **СИЗ** и **«В»** правила мягче (см. **FAQ**).  
- Поле **«Регистрационный номер»** в настройках может заполнять пробелы, если из файла реестра номер не найден.

### 4.4. Обновили выгрузку с сайта

1. Снова скачайте **.xlsx** с портала.  
2. Укажите **новый файл** в программе (или замените файл на диске и обновите путь / нажмите **F5**).  
3. **«Минтруд» → «Пересформировать протокол (номера реестра)…»** — пересоберёт протокол с теми же сотрудниками и отмеченными программами, заново прочитав реестр.

## Часть 5. Реквизиты и выгрузка для сайта Минтруда

1. **«Минтруд» → «Реквизиты работодателя…»** — **ИНН**, **наименование** работодателя, при необходимости **организация 2** (ИНН и наименование).  
2. В каталоге комплекта должен лежать XSD-шаблон Excel (см. часть 1, п. 8).  
3. **«Минтруд» → «Шаблон для загрузки на сайт…»** — выбор записей из **журнала** протоколов, сохранение заполненного **.xlsx**. **Рядом с тем же именем** автоматически создаётся файл **.xml** с теми же строками данных (удобно для контроля и обработки). **Формат и способ загрузки на портал** уточняйте по **актуальной инструкции АКОТ** — программа формирует файлы по заложенной логике и шаблону из комплекта.  
4. Папка сохранения по умолчанию — **`Mintrud/`** в каталоге данных программы (при необходимости выберите другую в диалоге).

---

## Часть 6. Заполнение протокола: главное окно

Выполняйте по порядку.

1. **Режим протокола**  
   - Обычный протокол по охране труда — галочка **«Технический протокол»** **снята**. Доступны подстановки из **реестра Минтруда** (если файл реестра указан).  
   - **«Технический протокол»** — отдельный шаблон Word и отдельная вкладка комиссии (**«Технич. вопросы»** в окне приказа). **Реестр Минтруда в этом режиме не используется.**

2. **Сотрудники**  
   - В списке выделите одного или нескольких (**Ctrl**, **Shift**).  
   - Поле **«Поиск»** сужает список (**Ctrl+F**).  
   - При необходимости введите **ФИО**, **должность**, **подразделение** вручную.

3. **Программы обучения**  
   - Отметьте **Б**, **ПП**, **СИЗ**, **В** по факту обучения.  
   - Для шаблона **Word** нужна **хотя бы одна** отмеченная программа (если не используете только текстовый шаблон).

4. **Дата** — дата протокола (**ДД.ММ.ГГГГ** или **ДД.ММ.ГГ**).

5. **№ протокола** — внутренний номер; в бланк Word часто подставляется сочетание **номер–месяц–год** из поля даты (см. переменные шаблона).

6. **Оценка** и **вид проверки** (плановая / внеплановая).

7. **«Сформировать протокол»**.

---

## Часть 7. Предпросмотр и проверка

1. Окно **«Предпросмотр протокола»** — **ФИО**, **таблица**, **номера из реестра** (если применимо), **темы программ**.  
2. **«Предпросмотр…»** на главном окне открывает **последний** сформированный текст.

---

## Часть 8. Сохранение готовых файлов

1. **«Сохранить в DOCX»** — папка по умолчанию часто **`Protokol/`** в каталоге данных программы.  
2. **«Сохранить в PDF»** — при **Word** на Windows PDF близок к документу; без Word — упрощённый PDF из предпросмотра.  
3. Дальше — печать, подписи, передача в электронном виде.

---

## Часть 9. Несколько протоколов сразу

Кнопка **«По одному на каждого → в папку…»** (только **Word**):

- для каждого выбранного сотрудника — отдельный **DOCX** и свой номер по порядку;  
- записи попадают в **журнал**;  
- предпросмотр — **последний** файл; остальные откройте из папки.

---

## Часть 10. Журнал и повторное использование

- В **«Настройки и данные…»**: **«Журнал (охрана труда)…»** и **«Журнал (тех. протоколы)…»** — раздельно для обычных и технических протоколов.  
- **Полный текст** протокола в базу **намеренно не сохраняется** (меньше персональных данных в SQLite); в журнале — метаданные и фрагменты для отчётности.  
- **Выгрузка Минтруда** (**«Шаблон для загрузки на сайт…»**) строится по **записям журнала** и сохранённым при формировании сведениям о программах и протоколе; **ФИО** и **СНИЛС** при выгрузке подтягиваются из **актуального Excel сотрудников**.

---

## Часть 11. Если что-то пошло не так

1. **«Обновить базы с диска»** или **F5**.  
2. Убедитесь, что **Excel** не держит файлы открытыми exclusively (сеть, «только для чтения»).  
3. **«Справка» → «Справка и FAQ…»** или **FAQ.md** в комплекте.  
4. **`protocol_errors_journal.txt`** — журнал ошибок (UTF-8).  
5. Для **.exe** не переносите один файл без папки — нужен **весь комплект** (шаблоны, **Data_base.xlsx** при необходимости, XSD-шаблон Минтруда).  
6. Требования портала Минтруда / АКОТ со временем меняются — сверяйтесь с **официальными материалами** на дату работы.
