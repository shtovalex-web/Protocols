# Подробная инструкция по программе ProtocolOOT

**Версия программы:** 1.4 (см. **«Справка» → «О программе…»**).  
**Для кого:** для тех, кто впервые открывает программу или редко с ней работает.  
**Задача программы простыми словами:** взять данные о людях и программах обучения из Excel, вставить их в готовый бланк протокола Word и сохранить готовый файл (Word или PDF).

Краткая версия — в файле **ИНСТРУКЦИЯ_для_пользователя.md**.  
Ответы на отдельные вопросы — в **FAQ.md** (меню **«Справка» → «Справка и FAQ…»**).

## 1. Что вы видите на экране

Представьте, что программа — это **конвейер**:

1. Слева вверху вы **подгружаете свежие данные** из Excel (кнопка обновления).
2. В большой рамке **«Формирование протокола»** вы говорите: *кого* проверяли, *по каким программам*, *какая дата и номер*.
3. Ниже — кнопки **«собрать»** документ и **«сохранить»** его в файл.
4. Вверху в полоске меню — **настройки**, **комиссия**, **Минтруд**, **справка**.

Заголовок окна: **ProtocolOOT — протоколы ОТ**.

**Порядок полей в рамке «Формирование протокола»** (сверху вниз): поиск → подразделение → список сотрудников → ФИО → должность → программы → дата и номер → оценка и вид проверки.

## 2. Что положить в папку с программой до работы

| Файл | Зачем он |
|------|----------|
| **Data_base.xlsx** | Список сотрудников (ФИО, должность, подразделение, СНИЛС). Часто там же лист **komission** — кандидаты в комиссию. |
| **Programs_base.xlsx** | Или отдельно: листы программ **B**, **V_PROF**, **PP**, **SIZ**, **V**. Можно держать в том же Data_base. |
| **default_protocol.docx** | Бланк обычного протокола Word. |
| **default_protocol_tehnicheskiy.docx** | Бланк для технического протокола (если нужен). |
| **Шаблон_Минтруд_XSD_УМН.xlsx** | Если будете выгружать данные на сайт Минтруда. |

Программа **сама создаст** в своей папке:

- **protocols.db** — журнал и настройки;
- **Protokol/** — сюда часто сохраняют готовые протоколы;
- **Mintrud/** — выгрузки для портала;
- **protocol_errors_journal.txt** — текст ошибок (если что-то сломалось).

**Важно:** если у вас программа в виде **одного файла exe**, переносите **всю папку** со сборкой, а не только exe.

## 3. Как запустить программу

**Способ 1 — exe:** дважды щёлкните **ProtocolOOT.exe** в папке программы.

**Способ 2 — Python:** откройте папку проекта и запустите **запуск.bat** или команду `py -3 main.py`.

Окно может появиться не мгновенно — подождите несколько секунд.

## 4. Блок «Обновить базы с диска»

**Где:** самая верхняя кнопка под логотипом, слева.

**Что делает:** заново читает с диска:

- файл с **сотрудниками**;
- файл с **программами обучения** (листы B, V_PROF, PP, SIZ, V, Tech_V и т.д.).

**Когда нажимать:**

- только что открыли программу;
- вы **сохранили изменения в Excel** (добавили человека, поправили должность);
- список сотрудников пустой или «старый».

**Горячая клавиша:** **F5** — то же самое.

**Если не нажать:** программа покажет то, что запомнила в прошлый раз. Новые строки из Excel не появятся.

**Частая ошибка:** Excel открыт на этом файле «только для чтения» в сети — закройте файл или снимите блокировку, потом снова **F5**.

## 5. Блок «Формирование протокола» — общее

Это **главная рамка** по центру окна. Здесь вы отвечаете на вопросы:

- *Кого* вносим в протокол?
- *По каким программам* проверяли?
- *Когда* и *под каким номером*?
- *Какая оценка* и *плановая или внеплановая* проверка?

**Логичный порядок работы:**

1. Обновить базы (**F5**).
2. Найти и выбрать людей в списке (при необходимости — уточнить подразделение).
3. Поставить галочки программ; для **«В»** при совмещениях — пройти диалог (см. §11).
4. Проверить дату и номер.
5. Нажать **«Сформировать протокол»**.
6. **«Сохранить в DOCX»** / **PDF**.

## 6. Поле «Поиск»

**Где:** первая строка внутри рамки «Формирование протокола».

**Что делает:** **фильтрует** длинный список сотрудников. Показывает только тех, у кого в данных есть введённый текст.

**Ищет по:** фамилии, имени, должности, подразделению, СНИЛС (что есть в Excel).

**Как пользоваться:**

1. Начните печатать фамилию, например `Иванов`.
2. Список ниже сузится (фильтр с небольшой задержкой, чтобы не тормозить при наборе).
3. Щёлкните нужную строку сотрудника.

**Совет:** не обязательно вводить полное ФИО — достаточно части слова.

**Горячая клавиша:** **Ctrl+F** — курсор сразу прыгнет в поле поиска.

**Частая ошибка:** в поиске остался старый текст — список «пустой». Очистите поле поиска и снова увидите всех.

## 7. Поле «Подразделение»

**Где:** строка **над** списком сотрудников (не под ФИО).

**Что делает:** подставляет **подразделение сотрудника** из Excel при выборе в списке; можно **изменить вручную** для этого протокола.

**Не путать** с полем **«Подразделение (место проверки знаний)»** в окне **«Приказ и комиссия»** — оно идёт в шаблон Word по маркеру **{{ПОДРАЗДЕЛЕНИЕ_ПРОВЕРКИ}}** (место проведения проверки для всего протокола). См. §20.

## 8. Список сотрудников

**Где:** большое поле со списком **под** полем «Подразделение».

**Что делает:** показывает людей из Excel, **сгруппированных по подразделениям**. Выбранные здесь попадут в протокол (в таблицу результатов и в текст).

### Группы по подразделениям

Строки вида **«▾ Название цеха (12)»** — это **заголовок группы**, а не сотрудник.

- Щёлкните по заголовку — группа **свернётся** или **развернётся**.
- В свёрнутой группе сотрудники не видны, пока снова не развернёте.

### Как выбрать одного человека

Щёлкните **один раз** по строке с ФИО — она подсветится.

### Как выбрать нескольких

- **Ctrl** — щёлкайте по строкам по одной (добавить/убрать из выбора).
- **Shift** — щёлкните первую строку, затем последнюю — выделится диапазон.

**Не выделяйте** строки-заголовки групп — только строки с людьми.

### Что происходит после выбора

Поля **ФИО**, **Должность**, **Подразделение** заполнятся сами из Excel (для **одного** выбранного; при нескольких — ФИО в поле может быть пустым, данные берутся из выбранных строк).

### Если списка нет совсем

1. Нажмите **F5**.
2. Откройте **«Администрирование» → «Настройки и данные…»** → **«Загрузить из Excel»** или **«Файл сотрудников…»**.
3. Проверьте, что в Excel есть лист с именем вроде **rabotnik**, **Работники**, **Сотрудники** (регистр не важен).
4. Первая строка листа — **заголовки** (ФИО, должность…).

### Несколько строк Excel на одно ФИО

Если у одного человека в базе **несколько строк** (основная и совмещаемая должности), при выборе всех таких строк программа учтёт **обе должности** — особенно для программы **«В»** (см. §11).

## 9. Поля «ФИО» и «Должность»

**Где:** под списком сотрудников.

### ФИО

**Когда заполняется само:** выбрали **одного** человека в списке.

**Когда вручную:** человека **нет в списке** — введите полностью: фамилия, имя, отчество.

### Должность

**Основная должность** для протокола:

- при **одном** выбранном — берётся из поля **«Должность»** на главном окне;
- при **нескольких** сотрудниках и программе **«В»** — для **каждого ФИО** своя основная должность задаётся в диалоге **«Совмещения — программы «В»»** (см. §11).

**Совмещаемая должность** (вторая строка в Excel) учитывается для шапки и таблицы программы **«В»**, если отмечена в диалоге совмещений.

**Подсказка под полем:** программа показывает, **нашла ли** должность на листе **V_PROF**. Если совпадений несколько — появится список **V_PROF:** и кнопка **«Подставить»** (похожие профессии по первым словам должности).

**Важно для программы «Б» и реестра Минтруда:** должность в протоколе и в Excel должна **совпадать** с реестром для блока **«Б»** (см. §21).

## 10. Блок «Программы обучения» (галочки)

**Где:** рамка с короткими подписями: **«Б»**, **ПП**, **СИЗ**, **«В»**.

| Галочка | О чём программа | Откуда программа берёт текст |
|---------|-----------------|------------------------------|
| **«Б»** | Базовая программа обучения по ОТ | Лист **B** и фрагменты по должности с листа **V_PROF** |
| **ПП** | Первая помощь | Листы **PP** и **V_PROF** (столбец для ПП) |
| **СИЗ** | Средства индивидуальной защиты | Листы **SIZ** и **V_PROF** (столбец для СИЗ) |
| **«В»** | Дополнительные программы по должности | Лист **V_PROF** (несколько столбцов с темами); лист **V** — ID в гос. реестре |

**Как выбрать:** отметьте **все программы**, по которым реально проверяли этого человека (или группу).

**Обязательно для Word:** нужна **хотя бы одна** галочка (если вы не используете особый текстовый шаблон .txt).

**Подсказка:** наведите мышь на галочку — всплывёт короткое пояснение.

### Несколько человек и несколько программ

Если выбрано **несколько** сотрудников и **несколько** программ, в таблице протокола появятся **строки для каждого** (логика зависит от шаблона Word).

## 11. Программа «В» и совмещения должностей

Диалог открывается **после нажатия «Сформировать протокол»**, если:

- отмечена галочка **«В»**;
- у **хотя бы одного** выбранного сотрудника в данных **больше одной должности** (основная и совмещаемая из Excel или несколько строк с одним ФИО).

**Заголовок окна:** «Совмещения — программы «В»».

### Что настроить в диалоге

Для **каждого ФИО** отдельно:

| Элемент | Значение |
|---------|----------|
| **Галочка** у должности | Учитывать эту должность в программах **«В»** (шапка и таблица) |
| **«основная»** (радиокнопка) | Какая должность попадёт в графу **«Должность»** протокола **для этого человека** |

**Обязательно:** у каждого сотрудника — **хотя бы одна** отмеченная должность и выбранная **основная** среди отмеченных.

**Отмена** в диалоге — формирование протокола **прерывается**.

### Если диалог не появился

- не отмечена **«В»**; или
- у всех выбранных **только одна** должность — используется поле **«Должность»** на главном окне (для одного человека) или данные из Excel.

### Нумерация в таблице «В»

Если в ячейке V_PROF текст начинается с **«2.»**, **«3)»** и т.п., в протоколе сохранится **тот же номер** (не всегда с единицы). Подробнее — **FAQ.md**.

## 12. Технический протокол (отдельный режим)

**Где:** галочка **«Технич. протокол (лист Tech_V)»** под программами.

**Это другой вид протокола** — не смешивайте его с обычным без необходимости.

### Когда включать

Проверка **технических** знаний по отдельному бланку и листу **Tech_V** в Excel.

### Что меняется

| Обычный протокол | Технический протокол |
|------------------|----------------------|
| Шаблон **default_protocol.docx** | Шаблон **default_protocol_tehnicheskiy.docx** (или свой) |
| Комиссия — вкладка «по проверке знаний» | Комиссия — вкладка **«Технич. вопросы»** |
| Можно подставлять номера из реестра Минтруда | Реестр Минтруда **не используется** |
| Галочки Б, ПП, СИЗ, В | Выбор **одной строки** из списка **Tech_V** |

### Как работать

1. Поставьте галочку **«Технич. протокол»**.
2. Появится список **«Программа по листу Tech_V»** — выберите нужную строку.
3. При необходимости: **«Шаблон Word (тех.)…»** / **«Сброс тех. шаблона»**.
4. Заполните ФИО, дату, номер — как в обычном режиме.
5. **«Сформировать протокол»**.

После **F5** список Tech_V обновляется с диска.

## 13. Дата, № протокола, Оценка, Проверка знаний

### Дата

**Формат:** `ДД.ММ.ГГГГ`, например `22.05.2026`.  
Можно `ДД.ММ.ГГ` — программа поймёт.

**Зачем:** дата в шапке протокола; из неё же берётся **месяц и год** для номера в Word (вид `5-05-2026` — номер-месяц-год).

### № протокола

**Число**, которое вы ведёте по журналу (1, 2, 3…).

Программа **запоминает** последний номер и подставит при следующем запуске.

В бланке Word часто получается строка: **ваш номер + месяц + год из поля «Дата»**.

### Оценка

- **удовлетворительно**
- **неудовлетворительно**

Попадает в таблицу результатов в Word.

### Проверка знаний

- **плановая**
- **внеплановая**

Тоже для таблицы в протоколе.

## 14. Кнопки «Сформировать протокол» и соседние

### «Сформировать протокол»

**Главная кнопка.** Она:

1. При программе **«В»** и совмещениях — открывает диалог (§11), если нужно.
2. Собирает документ Word по шаблону.
3. Открывает окно **«Предпросмотр»** (упрощённый текст).
4. Делает запись в **журнал** (метаданные в базе; полный текст в журнал **не сохраняется**).
5. **Включает** кнопки «Сохранить в DOCX», «Сохранить в PDF», «Предпросмотр».

**До нажатия** сохранить файл нельзя — сначала нужно сформировать.

**Если ошибка:** прочитайте сообщение на экране; при необходимости файл **protocol_errors_journal.txt** в папке программы.

### «По одному на каждого → в папку…»

**Только для Word.** Если выбрано **несколько** сотрудников:

- программа сделает **отдельный файл .docx на каждого**;
- номера протоколов **увеличиваются по порядку** (5, 6, 7… от числа в поле «№ протокола»);
- в предпросмотре останется **последний** документ;
- каждый сохранённый файл **можно сразу редактировать в Word** (защита с шаблона снимается автоматически).

### «Обновить протокол из реестра Минтруда»

**Когда:** вы уже сформировали протокол, потом скачали с сайта **новый** файл «Реестр обученных» с регистрационными номерами.

**Что делает:** пересобирает протокол **с теми же** ФИО, датой, программами, но **подставляет номера из реестра** в таблицу.

**Не работает** в режиме **технического** протокола.

Сначала укажите файл реестра: **«Минтруд» → «Файл реестра обученных»** (см. §21).

## 15. Кнопки «Сохранить в DOCX», «PDF», «Предпросмотр»

**Серые (неактивные)?** Сначала нажмите **«Сформировать протокол»**.

### «Сохранить в DOCX»

Сохраняет документ **Word** (.docx).  
Обычно предлагается папка **Protokol** рядом с программой.

Имя файла часто строится из номера, даты и фамилии.

**Редактирование:** готовый протокол сохраняется **без защиты Word** (шаблоны в папке программы могут быть защищены от случайной правки — это не относится к сформированным файлам).

### «Сохранить в PDF»

Сохраняет **PDF**.

**На Windows «как в Word»:** нужен установленный **Microsoft Word**. Программа собирает docx и сохраняет PDF через Word.

**Если Word нет:** может сохраниться **упрощённый** PDF из текста предпросмотра (таблица может выглядеть проще).

### «Предпросмотр…»

Снова открывает окно с текстом **последнего** сформированного протокола (если вы его закрыли).

---

## 16. Окно «Предпросмотр протокола»

**Зачем:** посмотреть, **что получилось**, до сохранения в файл.

**Что там есть:**

- большое текстовое поле (это **упрощённый** вид, не 100% как в Word);
- те же кнопки **сохранить DOCX / PDF**;
- можно закрыть крестиком — на главном окне протокол **не пропадёт**, пока вы не сформируете новый.

**Важно:** окончательный вид таблиц и шрифтов — в сохранённом **.docx**, откройте его в Word.

## 19. Окно «Настройки и данные» — по блокам

Откройте: **«Администрирование» → «Настройки и данные…»**.

### Блок «Файл данных (сотрудники)»

| Кнопка | Действие |
|--------|----------|
| **Загрузить из Excel** | Прочитать сотрудников в список на главном окне |
| **Файл сотрудников…** | Указать **другой** файл вместо Data_base.xlsx |

Подпись внизу покажет, **какой файл** сейчас выбран.

**Лист komission** в том же файле — кандидаты в комиссию (см. §20). Сам состав комиссии задаётся в окне **«Приказ и комиссия»**.

### Блок «Справочник программ обучения»

| Кнопка | Действие |
|--------|----------|
| **Файл программ…** | Указать **Programs_base.xlsx** |
| **Вынести листы из Data_base…** | Помочь разделить старый общий файл на два |

Если отдельного файла программ нет — программа читает листы B, V_PROF… из **того же** файла, что и сотрудники.

### Блок «Шаблон протокола»

| Кнопка / элемент | Действие |
|------------------|----------|
| **Выбрать шаблон** | Ваш .docx или .txt вместо стандартного |
| **Переменные шаблона** | Справка: **куда** программа подставляет номер, дату, таблицу, комиссию |
| **Защита стандартных шаблонов в Word** (галочка) | Автоматически защищать **шаблоны** в папке программы от правки |
| **Включить защиту шаблонов…** | Принудительно включить режим «только чтение» в Word на default_protocol*.docx |
| **Снять защиту (для правки)…** | Снять защиту с шаблонов, чтобы править бланк |
| **Папка шаблона…** | Открыть папку, где лежит текущий шаблон |
| **Шаблон Word (тех.)…** / **Сброс тех. шаблона** | Бланк технического протокола |

**Важно:** **сформированные** протоколы при сохранении **не остаются защищёнными** — их можно править в Word сразу после сохранения.

### Блок «Реестр обученных»

Файл **Excel с портала Минтруда** после регистрации проверки.

| Кнопка | Действие |
|--------|----------|
| **Выбрать файл…** | Указать выгрузку .xlsx |
| **Сбросить** | Забыть путь к файлу |

Нужны колонки вроде: **Номер в реестре**, **ФИО** (или Фамилия/Имя/Отчество), **СНИЛС**, программа, номер протокола.

### Блок «Дополнительно при заполнении»

| Поле | Зачем |
|------|-------|
| **Регистрационный номер** | Если в реестре номер **не нашёлся** — подставится этот (запасной) |
| **Доп. тема / строка для .txt** | Для текстового шаблона, не для обычного Word |

### Кнопки журнала

| Кнопка | Зачем |
|--------|-------|
| **Журнал (охрана труда)…** | Список сформированных обычных протоколов |
| **Журнал (тех. протоколы)…** | То же для технических |
| **Очистить все журналы…** | Удалить историю (осторожно!) |

### Окно журнала (подробнее)

В журнале хранятся **не полный текст протокола**, а **ФИО, дата, оценка, № протокола, тема/программы**, время записи. Полный текст в базе **намеренно не сохраняется** (меньше персональных данных).

| Кнопка в журнале | Действие |
|------------------|----------|
| **Обновить список** | Перечитать записи из базы |
| **Выгрузить реестр…** | Excel или CSV со списком сформированных протоколов |
| **В окно предпросмотра** | Только если в старой записи был сохранён текст; иначе — сформируйте снова или откройте DOCX |
| **Очистить журнал** | Удалить записи этого вида (ОТ или тех.) |

**Выгрузка реестра:** если в одной записи журнала несколько ФИО (протокол на группу), в файле будет **отдельная строка на каждого** (данные берутся из сохранённого снимка сотрудников или из поля ФИО через запятую/точку с запятой).

## 20. Окно «Приказ и комиссия»

**«Администрирование» → «Приказ и комиссия…»**.

Две вкладки:

1. **По проверке знаний работников** — обычный протокол.
2. **Технич. вопросы** — технический протокол.

Данные **разные** — заполните обе, если пользуетесь обоими видами протоколов.

### Поля приказа и шапки

| Поле | Что вписать |
|------|-------------|
| **№ приказа о комиссии** | Номер приказа, например `125` |
| **Дата приказа** | `ДД.ММ.ГГГГ` |
| **Подразделение (место проверки знаний)** | Текст для **{{ПОДРАЗДЕЛЕНИЕ_ПРОВЕРКИ}}** в Word (цех, участок, «для работников…») |
| **В соответствии с приказом (кем утверждён…)** | Фрагмент для **{{УТВЕРДИЛ_ПРИКАЗ}}** (без слов «в соответствии с приказом») |

Маркеры **{{ПОДРАЗДЕЛЕНИЕ_ПРОВЕРКИ}}** и **{{УТВЕРДИЛ_ПРИКАЗ}}** не удаляйте из шаблона .docx.

### Список кандидатов (середина окна)

Люди с листа **komission** в Excel: строка 2 — заголовки; с 3-й строки — председатель в A/B, члены в D/E.

| Кнопка | Действие |
|--------|----------|
| **Обновить список с Excel** | Перечитать лист komission |
| **Назначить председателем** | Выбранный кандидат → председатель |
| **Добавить в члены комиссии** | Выбранный кандидат → в члены |

### Председатель и члены

- **Председатель** — один человек.
- **Члены** — список; можно **удалить** выбранного или **очистить** весь список.

### Сохранить

Кнопка **«Сохранить в базу»** — запомнить состав. **Пока не сохранили** — в протоколе могут быть старые данные.

В Word в шапке:

- после слова **«председателя»** — ФИО и должность (родительный падеж);
- после **«членов»** — список **в два столбца**, с **подчёркиванием** имён и должностей.

В конце бланка могут быть плейсхолдеры **{{ПРЕДСЕДАТЕЛЬ}}** и **{{ЧЛЕНЫ_КОМИССИИ}}** — блоки для подписей.

**Вставка из буфера:** в многострочных полях и в полях ввода работают **Ctrl+V** и **ПКМ → «Вставить»**.

## 21. Меню «Минтруд»

| Пункт | Зачем |
|-------|-------|
| **Реквизиты работодателя…** | ИНН, название и др. для файла выгрузки на сайт |
| **Файл реестра обученных (.xlsx)…** | Файл с номерами после регистрации на портале |
| **Пересформировать протокол (номера реестра)…** | То же, что кнопка на главном окне |
| **Шаблон для загрузки на сайт…** | Заполнить **Шаблон_Минтруд_XSD_УМН.xlsx** по журналу; рядом создаётся **.xml** |

### Типовая цепочка с Минтрудом

1. Сформировали протоколы, провели проверку, зарегистрировали на портале.
2. Скачали **реестр обученных** (.xlsx).
3. Указали файл в программе.
4. **«Обновить протокол из реестра Минтруда»** — номера в таблице Word.
5. Для загрузки на портал: **«Шаблон для загрузки на сайт…»**, сохранить xlsx и xml.

Подробнее про портал — **ИНСТРУКЦИЯ_оформление_протоколов_Минтруд.md**.

### Как программа ищет номер в реестре (коротко)

1. Сначала **СНИЛС** (если есть).
2. Иначе **ФИО** + номер протокола + программа.
3. Для программы **«Б»** при нескольких строках важно **совпадение должности** в реестре.
4. Для **ПП**, **СИЗ**, **«В»** должность в реестре не обязательна, но помогает выбрать строку при нескольких совпадениях.

## 23. Один полный пример: от начала до файла на диске

**Задача:** протокол для Иванова И.И., программы «Б» и ПП, дата сегодня, номер 42.

1. Запустить программу.
2. **F5** — обновить базы.
3. В **поиске** набрать `Иванов`, в списке щёлкнуть **Иванов И.И.** (при необходимости развернуть группу подразделения).
4. Проверить **Подразделение** и **Должность**.
5. Поставить галочки **«Б»** и **ПП**.
6. **Дата** — сегодняшняя. **№ протокола** — `42`.
7. **Оценка** — удовлетворительно. **Проверка** — плановая.
8. **«Приказ и комиссия»** — председатель, члены, место проверки → **Сохранить в базу** → закрыть.
9. **«Сформировать протокол»** — прочитать предпросмотр.
10. **«Сохранить в DOCX»** — папка **Protokol** → сохранить.
11. Открыть файл в Word — документ **доступен для правки**; проверить таблицу и подписи.
12. При необходимости **«Сохранить в PDF»**.

Готово.

## 24. Частые ошибки и что делать

| Симптом | Что проверить |
|---------|----------------|
| Список сотрудников пуст | **F5**; путь к Excel в настройках; имя листа; закрыт ли Excel |
| После поиска «0 из N» | Очистите поле **Поиск** |
| «Нужна хотя бы одна программа» | Поставьте галочку **«Б»**, **ПП**, **СИЗ** или **«В»** |
| Диалог «Совмещения» — не даёт продолжить | У каждого ФИО отметьте должность и выберите **основную** |
| Таблица в Word пустая | В шаблоне есть маркеры **1.1**, **2.1** и слово **ФИО** во 2-й колонке — см. **«Переменные шаблона»** |
| Номер из реестра не подставился | Указан файл реестра? Совпадают СНИЛС/ФИО/№ протокола? Для «Б» — должность? |
| Word не даёт править **шаблон** | Это нормально — **«Снять защиту (для правки)…»** в настройках; **готовый протокол** не защищён |
| PDF «странный» | Установлен ли **Word**? Иначе сохранится упрощённый PDF |
| Ctrl+V не вставляет | Кликните в поле ввода; **ПКМ → Вставить**; перезапустите программу |
| Ошибка при сохранении Excel Минтруда | Файл **Шаблон_Минтруд_XSD_УМН.xlsx** в папке программы |
| После переноса exe «ничего не находит» | Копируйте **всю папку** сборки, не один exe |
| В журнале нет текста протокола | Так задумано — откройте сохранённый DOCX или сформируйте снова |

Технические подробности и сборка — **README.md** (для IT-специалиста).
