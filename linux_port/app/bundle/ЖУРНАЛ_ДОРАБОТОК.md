# Журнал доработок ProtocolOOT

История изменений по версиям. Нумерация ведётся с **1.4**.  
Текущая версия в программе: **«Справка» → «О программе…»** (`APP_VERSION` в `ProtocolOHT_next/protocol_app_info.py`).

**Правило:** при каждом изменении `APP_VERSION` добавляйте раздел ниже с датой и перечнем доработок.

---

## 1.5 — июнь 2026

### Журнал протоколов
- При повторном формировании протокола с тем же **№, датой и видом** запись в `protocols.db` **обновляется**, а не дублируется.
- При запуске программы **удаляются старые дубликаты** в журнале (остаётся запись с наибольшим `id` — актуальная).
- В окне **«Журнал протоколов»** и в окне **«Шаблон для сайта Минтруда»** в списке показывается **одна строка на протокол** (без повторов).

### Выгрузка Минтруд
- Excel/XML формируются **только из выбранных записей** журнала.
- Убрано автоматическое **слияние с уже сохранённым `.xlsx`** — оно приводило к лишним строкам при повторной выгрузке.
- Внутри одной выгрузки по-прежнему убираются дубли строк (один сотрудник × программа × протокол).

### Комиссия по проверке знаний
- **Именованные профили** комиссии (например «НПС и ЦТТ») — отдельно для охраны труда и для технических вопросов.
- Профили хранятся в таблице `commission_profiles` в `protocols.db`: приказ, место проверки, состав.
- В окне комиссии: выбор профиля из списка, **Загрузить**, **Сохранить** (создание/обновление), **Удалить профиль**.
- При первом запуске после обновления прежняя комиссия из настроек переносится в профиль «Основная» (или по названию подразделения).

### Прочее
- Добавлены unit-тесты: журнал (upsert, очистка дублей), профили комиссии, dedupe строк выгрузки.
- Меню **«Справка» → «Журнал доработок…»**; файл `ЖУРНАЛ_ДОРАБОТОК.md` в комплекте.
- При первом запуске после обновления — сообщение об удалении старых дублей журнала (один раз на версию).
- Партия «по одному на каждого» обновляет журнал без предварительного удаления строк.
- Версия в «О программе»: **1.5**.

## 1.4 — июнь 2026

Базовая зафиксированная версия, с которой ведётся этот журнал.

### Документация
- **Подробная инструкция** пользователя (`ПОДРОБНАЯ_ИНСТРУКЦИЯ_для_пользователя.md` / `.docx`) с **19 скриншотами** интерфейса.
- Скрипты: `tools/capture_manual_screenshots.py`, `tools/instruction_md_to_docx.py`, `generate_podrobnaya_instruction_docx.bat`.

### Минтруд и протоколы
- Выгрузка для реестра Минтруда: программа **«В»** — должности и программы из **метаданных протокола** (снимок при сохранении в журнал).
- Параллельно сохраняются **`.xlsx` и `.xml`** с одними данными.
- Меню **«Минтруд»**: реквизиты работодателя, выбор записей из журнала для шаблона Excel.

### Интерфейс и шаблоны
- Обновлены шаблоны `default_protocol.docx` и `default_protocol_tehnicheskiy.docx`.
- Окно **«О программе»**: кликабельный **mailto** с темой письма для обратной связи.
- С **сформированных DOCX** снимается защита редактирования (удобнее править в Word).

### Сборка и перенос
- **Linux**: CI-сборка PyInstaller, ветка `linux`, `tools/sync_linux_branch.py`.
- Обновление **эталона** переносимой сборки (`tools/update_etalon.py`).

### Версия
- В «О программе»: **1.4**.
